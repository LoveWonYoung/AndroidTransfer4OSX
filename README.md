# AndroidTransfer4OSX

一个基于 Go + Fyne 的 macOS 桌面工具，用于通过 ADB 将本地文件/目录批量传输到 Android 设备。

## 功能特性

- 图形界面操作，无需手敲完整 adb 命令。
- 自动刷新并识别已连接设备（`adb devices -l`）。
- 支持选择常用目标目录（内置存储/TF 卡目录识别）。
- 支持单个文件、单个目录、多文件/多目录队列顺序传输。
- 支持拖拽文件或目录到窗口直接加入队列。
- 支持任务取消、日志查看与清空。
- 支持传输速度显示：
  - 单项传输完成后在队列状态、状态栏和日志中显示速度。
  - 全部任务完成后显示总体平均速度与总传输数据量（可统计时）。

## 环境要求

- macOS
- Go 1.24+
- Android Platform Tools（包含 `adb`）
- Android 设备开启 USB 调试，并完成授权

## 快速开始

### 1. 克隆并进入项目

```bash
git clone <your-repo-url>
cd AndroidTransfer4OSX
```

### 2. 运行

```bash
go run .
```

### 3. 打包

```bash
go build -o at4m .
```

## 使用说明

1. 启动应用后，在 `ADB 路径` 填入：
   - `adb` 可执行文件路径，或
   - `platform-tools` 目录路径。
2. 点击 `刷新设备`，选择目标 Android 设备。
3. （可选）点击 `识别存储`，从常用目录下拉框选择目标目录。
4. 通过以下任一方式加入传输队列：
   - `添加文件`
   - `添加目录`
   - 输入本地路径后点击 `加入队列`
   - 直接拖拽到窗口
5. 点击 `开始传输`。
6. 传输过程中可点击 `取消任务`。

## 速度显示说明

- 如果 ADB 输出包含速度统计（如 `MB/s`），界面会优先显示 ADB 提供的速度。
- 如果 ADB 未返回速度但可统计本地数据量，界面会按 `数据量 / 用时` 计算平均速度。
- 对于无法统计大小的特殊文件，速度会显示为 `未知`。

## 常见问题

### 1) 识别不到设备

- 确认 USB 线支持数据传输。
- 确认设备已开启 USB 调试。
- 在手机上确认 USB 调试授权弹窗。
- 终端执行 `adb devices` 检查是否有 `device` 状态。

### 2) ADB 路径无效

- 检查填写路径是否存在。
- 若填写的是目录，请确认目录下有 `adb` 可执行文件。

### 3) 传输超时或失败

- 检查设备连接稳定性。
- 检查目标目录是否有写入权限。
- 观察日志面板中的 ADB 输出详情。
